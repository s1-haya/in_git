今から git を学びます。
git add
git add .　全てのファイルをステージングエリアに保存
git commit -v 変更内容確認可能

git rm ファイル削除（リポジトり、ワークツリー）
git rm --cached ＜ファイル名＞　ファイル削除（リポジトリ）
git status gitリポジトリの現在の状況を確認
git log 変更履歴の確認
　●git log --oneline 一行で表示（何が変更されたかわかるため）
　●git log -p index.html ファイルの変更差分を表示（変更前と後を確認するため） 
　●git log -n int int分の最新のファイルの変更差分を表示

git diff ステージとローカルリポジトリとの差分
git diff HEAD ステージとコミットとの差分

リモートリポジトリ　＝ git hub
git remote add orign url  ローカルリポジトリからリモートリポジトリーに登録
git push orign master ローカルリポジトリからリモートリポジトリーに送信
git push orign feature master以外のブランチへの送信

git merge 
コンフリクト　複数人が同じ箇所を変更した場合に起きる現象。
